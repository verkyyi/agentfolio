# PDF Export Design

## Summary

Add PDF export to AgentFolio using the JSON Resume ecosystem. Each adapted JSON Resume file (`data/adapted/{slug}.json`) gets a corresponding PDF (`data/adapted/{slug}.pdf`) generated by `resumed` with `jsonresume-theme-developer-mono`. A dedicated GitHub Actions workflow handles generation, and the site renders a download button when a PDF is available.

## Architecture

### Pipeline

```
adapt.yml:  data/input/** changes → generates JSON → commits to data/adapted/
pdf.yml:    data/adapted/*.json changes → generates PDFs → commits to data/adapted/
```

PDF generation is fully decoupled from adaptation. The PDF workflow triggers on any change to `data/adapted/*.json`, regardless of source.

### PDF Generation

- **Tool:** `resumed` (npm package) — lightweight JSON Resume renderer
- **Theme:** `jsonresume-theme-developer-mono` — matches the visual feel of the site's ResumeTheme
- **Input:** `data/adapted/{slug}.json`
- **Output:** `data/adapted/{slug}.pdf`
- **Selective generation:** Only regenerate PDFs for changed JSON files (same pattern as adapt.yml)

### Workflow: `.github/workflows/pdf.yml`

**Triggers:**
- `push` to `main` with changes to `data/adapted/*.json`
- `workflow_dispatch` with optional `slugs` input (space-separated, empty = all)

**Steps:**
1. Checkout
2. Detect changed JSON files (extract slugs from changed paths; workflow_dispatch uses input or all)
3. Install `resumed` and `jsonresume-theme-developer-mono`
4. For each slug, run `resumed render data/adapted/{slug}.json --theme developer-mono -o data/adapted/{slug}.pdf`
5. Show diff / list generated PDFs
6. Commit and push (`chore(pdf): regenerate PDFs`)

**Committer:** AgentFolio Bot (same as adapt.yml)

### Download Button

- Small download icon/link in the top-right corner of the resume page
- Fetches `data/adapted/{slug}.pdf` — if 404, button does not render
- No error state, no loading spinner needed
- Minimal visual footprint — should not distract from resume content

### Local Development

`npm run copy-data` already syncs `data/adapted/` → `web/public/data/adapted/`, so locally generated PDFs are available at `/data/adapted/{slug}.pdf` without extra config.

## File Changes

| File | Change |
|------|--------|
| `.github/workflows/pdf.yml` | New workflow |
| `web/src/components/ResumeTheme.tsx` | Add download button |
| `web/src/hooks/useAdaptation.ts` | Check PDF availability |
| `data/adapted/*.pdf` | Generated artifacts, committed like JSON |

## Decisions

- **PDFs are committed** to the repo alongside JSON, same as adapted JSON files. They are generated artifacts.
- **Theme choice:** `jsonresume-theme-developer-mono`. If it doesn't produce acceptable output, swap the theme — no architectural changes needed.
- **No server-side rendering.** No headless browser. No changes to the Vite build.

## Out of Scope

- Custom JSON Resume theme matching the site pixel-for-pixel
- Print CSS improvements to ResumeTheme.tsx
- Serverless / on-demand PDF generation API
